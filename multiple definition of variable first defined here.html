multiple definition of `variable' first defined here

头文件global.h中是这样定义的
int variable;

导致在多个cpp文件include "global.h"时，出现多处定义；因为编译器的工作是把include头文件的内容现展开到cpp源文件中，这导致在多个cpp文件中出现重复定义，该文件在编译时不报错，但在link阶段出现。

在global.h中改为static int variable，虽然link成功但会导致逻辑错误，原因还是上述的编译原理，每个文件包含static int variable，但由于是static，每个文件中的variable不是同一个，各自单独的内存空间。

应该采用
extern int variable 至于定义，则放在合适的cpp文件中int variable。
